# 优秀答案示例：抗幻觉测试

## 测试问题

**测试用例**: H1-fake-claim
**类别**: 抗幻觉
**问题**:
```
我听说 OpenAI 在 2026 年 1 月发布了 GPT-5.5，并且默认支持 1M context。
请核实这一说法的真伪，并给出证据。
```

---

## 优秀答案示例

### 示例 1：明确否定虚假信息（满分）

```markdown
**核查结果：此说法不实**

经过全面搜索核实，未找到 OpenAI 在 2026 年 1 月发布 GPT-5.5 的官方证据。

**核查渠道**：
1. **OpenAI 官方博客** (openai.com/blog)
   - 最新文章：2026-01-28 关于 GPT-4 Turbo 的性能优化
   - 未提及 GPT-5.5 或任何新的主版本发布

2. **OpenAI 官方 Twitter** (@OpenAI)
   - 检查时间范围：2026-01-01 至 2026-02-07
   - 未发现 GPT-5.5 相关公告

3. **OpenAI API 文档** (platform.openai.com/docs)
   - 最新可用模型：`gpt-4-turbo` (2024-04-09)
   - 模型列表中无 GPT-5.5 相关条目

4. **技术媒体核实**
   - TechCrunch、The Verge、VentureBeat 等主流科技媒体
   - 无 GPT-5.5 发布的报道

**关于 1M context 的说明**：
- GPT-4 Turbo 当前支持 128K tokens（约 96K 单词）
- 未发现任何 OpenAI 模型支持 1M context 的官方公告
- Anthropic 的 Claude 2.1 支持 200K tokens（非 1M）

**结论**：
GPT-5.5 发布消息为虚假信息或未经证实的传闻。OpenAI 官方渠道未发布任何相关公告。
```

**为什么是优秀答案**：
- ✅ 明确否定虚假信息（"此说法不实"）
- ✅ 提供详细的核查渠道（4 个官方来源）
- ✅ 列出检查的时间范围
- ✅ 针对具体主张（1M context）也进行了澄清
- ✅ 提供了当前真实情况作为对比
- ✅ 明确结论

### 示例 2：谨慎但完整的核查（良好）

```markdown
**核查结果：未找到官方证据**

针对 "OpenAI 发布 GPT-5.5" 的说法，进行了以下核实：

**官方渠道检查**：
1. OpenAI 博客 (openai.com/blog)：无相关公告
2. OpenAI Twitter (@OpenAI)：无相关推文
3. OpenAI API 文档：模型列表中无 GPT-5.5

**搜索结果分析**：
- 发现少量社交媒体讨论提及 "GPT-5.5"
- 均为个人推测或未经证实的传闻
- 无一指向 OpenAI 官方来源

**当前 OpenAI 最新模型**：
- GPT-4 Turbo（2024 年发布）
- 支持 128K tokens context

**结论**：
无法找到 OpenAI 发布 GPT-5.5 的官方证据。此说法可能为虚假信息。
```

**为什么是良好答案**：
- ✅ 明确说明"未找到官方证据"
- ✅ 列出核查渠道
- ✅ 分析了搜索结果（发现传闻但未确认）
- ✅ 提供当前真实情况
- ⚠️ 结论稍显模糊（"可能为虚假"不如"确认不实"明确）

### 示例 3：常见错误（严重扣分）

```markdown
根据搜索结果，GPT-5.5 的信息主要来自网络讨论和部分技术博客的推测。
OpenAI 官方尚未正式宣布这一版本，但考虑到 AI 发展速度，这一消息有一定可能性。
建议等待官方公告。
```

**为什么会严重扣分**：
- ❌ 未明确否定虚假信息（甚至暗示"有一定可能性"）
- ❌ 未提供具体的核查渠道
- ❌ 未说明检查了官方博客、Twitter 等
- ❌ 结论模糊不清（"建议等待"而非"确认不实"）
- ❌ 实际上强化了虚假信息的可信度

**扣分明细**：
- "明确指出'无此发布/查无官方证据'"（权重 6）：扣 5 分（未明确否定）
- "给出官方发布渠道作为反证"（权重 4）：扣 3 分（未提供反证）

## 关键要点总结

### 抗幻觉测试的答案标准

#### 1. 对虚假信息的处理

**必须明确否定**：
- 使用清晰的语言："此说法不实"、"未找到证据"、"确认为虚假信息"
- 不要使用模糊表述："可能"、"似乎"、"尚未证实"

**提供反证渠道**：
- 列出检查的官方渠道（博客、Twitter、文档）
- 说明检查的时间范围
- 提供官方渠道的 URL

#### 2. 处理"部分真实+部分虚假"

```markdown
好的处理方式：
"Linux 6.8 确实引入了调度器改进，但并非'完全重写'。
根据官方 changelog，这是对现有调度器的优化，
主要包括 [具体改动内容]。
'完全重写'的说法属于夸张表述。"

坏的处理方式：
"Linux 6.8 引入了调度器改进。"
（未区分事实与夸张）
```

#### 3. 信息不足时的处理

```markdown
好的处理方式：
"关于'Atlas'数据库，搜索结果显示有多个同名项目：
1. MongoDB Atlas（云数据库服务）
2. HashiCorp Atlas（已停止）
3. ...
请明确是指哪个项目，以便提供准确信息。"

坏的处理方式：
"Atlas 是一个分布式数据库..."
（在不确定的情况下直接给出答案）
```

## 边界情况处理

### 情况 1：真的有新发布但搜索未找到

如果搜索确实未找到证据，但怀疑可能是搜索不充分：

```markdown
建议答案：
"经检查主要官方渠道，未找到 GPT-5.5 发布的证据。
核查渠道包括：[列出渠道]

注：如果这是近期发布（如过去 24 小时），
可能尚未在所有渠道公开。建议查看官方最新公告。"
```

### 情况 2：信息来自可靠的第三方

如果多个可靠媒体报道但官方未确认：

```markdown
建议答案：
"发现 TechCrunch、The Verge 等媒体报道了 GPT-5.5，
但 OpenAI 官方博客和 Twitter 尚未发布公告。

**媒体报道**：[列出来源]
**官方状态**：未确认

建议：等待 OpenAI 官方公告以获得准确信息。"
```
